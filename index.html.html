System Overview & IDS Engine
Features · Detection · Architecture
Core Capabilities
Everything the system does
A fully self-contained platform covering detection, AI analysis, response, and audit — no external APIs or databases required.
🛡️

Rule-Based IDS Engine

Fifteen weighted detection rules covering DDoS floods, brute-force login attacks, port scanning, and traffic spikes. A behavioral multiplier (1× to 3×) escalates repeat offenders automatically over a 60-second window.

ids_engine.py
🤖

AI Anomaly Detection

Pure-Python Isolation Forest with 100 trees, trained on 500 normal baseline packets. Scores each event 0–1 using 8-dimensional feature vectors. Adds up to +25 risk boost for confirmed anomalies.

anomaly_detector.py
📊

Multi-Factor Risk Scoring

Five-component scoring: base classification, attack severity, failed login factor (logarithmic), source frequency (exponential decay), and traffic abnormality index. Range: 0–100.

risk_assessment.py
⚡

Automated Response System

Four-tier response playbook. LOW logs silently. CRITICAL activates emergency mode, blocks the source IP, triggers service failover, and generates a formal incident report automatically.

response_system.py
🌍

Geo-Location Intelligence

Deterministic IP-to-country mapping — same IP always resolves to the same location. Tracks attack frequency per country and flags known hostile IP prefixes on first contact.

geo_locator.py
🔐

Secure Authentication

PBKDF2-SHA256 with 260,000 iterations and 32-byte random salt per user. Constant-time comparison via hmac.compare_digest(). Five-attempt brute-force lockout with 60s cooldown.

auth.py

Data Security
Authentication &
Credential Protection
Every component of the authentication system follows current security best practices — from password storage to session lockdown.
🔑

PBKDF2-SHA256 — 260,000 Iterations

OWASP-recommended minimum. Derives a 256-bit key from password + unique salt. Makes brute-force attacks computationally infeasible.

🧂

32-Byte Cryptographic Salt per User

Generated via secrets.token_bytes(32). Every user has a unique hash — eliminates rainbow table and pre-computation attacks entirely.

⏱️

Constant-Time Comparison

hmac.compare_digest() eliminates timing side-channel attacks — comparison time is always identical regardless of match position in the string.

🔒

Brute-Force Lockout — 60s Cooldown

5 failed attempts → 60-second lockout with a persisted counter. Lockout state survives process restarts via the JSON credential store.
